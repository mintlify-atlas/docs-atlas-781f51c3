---
title: 'Saved color history with localStorage auto-persistence'
description: 'Color Flipper automatically saves up to 20 colors and gradients in localStorage so your palette survives page reloads and browser restarts.'
---

Every color or gradient you generate or save is appended to a history list, displayed as a scrollable grid of swatches in the sidebar. The list is persisted to `localStorage` under the key `"cf_history"` so your work survives a page reload. Clicking any swatch instantly restores that color or gradient — including switching back to the correct mode.

## Initializing from localStorage

The history state is initialized lazily using a function passed to `useState`. This runs once on mount and reads any previously stored value before falling back to an empty array:

```jsx ColorFlipper.jsx
const [history, setHistory] = useState(() => {
  try {
    return JSON.parse(localStorage.getItem("cf_history")) || [];
  } catch {
    return [];
  }
});
```

## Persisting history changes

A `useEffect` writes the latest history array to `localStorage` every time it changes. Only the first 20 items are written:

```jsx ColorFlipper.jsx
useEffect(() => {
  try {
    localStorage.setItem("cf_history", JSON.stringify(history.slice(0, 20)));
  } catch {
    // ignore storage errors
  }
}, [history]);
```

## Adding items with `pushHistory`

`pushHistory` prepends the new item, deduplicates it (removes any earlier occurrence of the same value), and trims the list to 20 entries:

```jsx ColorFlipper.jsx
function pushHistory(item) {
  setHistory((prev) => {
    const next = [item, ...prev.filter((x) => x !== item)];
    return next.slice(0, 20);
  });
}
```

<Note>
  The history list is capped at 20 items. When the limit is reached, the oldest entry is dropped to make room for the newest one.
</Note>

## Restoring a history item

The `History` component renders each entry as a clickable swatch button. Clicking one calls the internal `useHistoryItem` function, which inspects the stored string to decide whether to restore a single color or a gradient:

```jsx History.jsx
function useHistoryItem(item) {
  if (!item) return;
  if (item.startsWith("linear-gradient")) {
    const matches = item.match(/#([0-9a-fA-F]{6})/g) || [];
    if (matches.length >= 2) {
      setMode("gradient");
      setColorA(matches[0]);
      setColorB(matches[1]);
    }
  } else if (item.startsWith("#")) {
    setMode("single");
    setColorA(item);
  } else {
    const m = item.match(/#([0-9a-fA-F]{6})/);
    if (m) {
      setMode("single");
      setColorA(m[0]);
    }
  }
}
```

The restoration logic follows these steps:

<Steps>
  <Step title="Detect the item type">
    If the stored string starts with `"linear-gradient"`, it is treated as a gradient. If it starts with `"#"`, it is a plain hex color.
  </Step>
  <Step title="Extract hex values">
    For gradients, a regex extracts up to two six-digit hex codes from the string. For single colors, `colorA` is set directly.
  </Step>
  <Step title="Switch mode and apply colors">
    `setMode` is called with `"gradient"` or `"single"`, then `setColorA` (and `setColorB` for gradients) are updated — immediately reflecting the restored values in the preview.
  </Step>
</Steps>

## Clearing history

The **Clear History** button in the Controls panel calls `clearHistory`, which empties the state array and removes the `localStorage` key:

```jsx ColorFlipper.jsx
function clearHistory() {
  setHistory([]);
  try {
    localStorage.removeItem("cf_history");
    showToast("History cleared!");
  } catch {}
}
```

A toast notification confirms the action. See [clipboard and toast notifications](/features/clipboard) for how the toast works.
